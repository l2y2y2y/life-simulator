# 《人生模拟器》UI设计方案文档

## 一、当前UI问题诊断

### 1.1 视觉风格问题
| 问题 | 具体表现 | 严重程度 |
|------|---------|---------|
| 色彩单调 | 整体暗色调，缺乏层次感和视觉焦点 | 高 |
| 缺乏品牌感 | 没有独特的视觉识别，像普通网页 | 高 |
| 字体单一 | 全部使用系统默认字体，无特色 | 中 |
| 圆角/阴影过度统一 | 所有卡片使用相同样式，缺乏变化 | 中 |

### 1.2 交互体验问题
| 问题 | 具体表现 | 严重程度 |
|------|---------|---------|
| 反馈不足 | 按钮点击、选择后缺乏动效反馈 | 高 |
| 信息密度不均 | 首页空旷，游戏页拥挤 | 中 |
| 缺乏引导 | 新玩家不知道下一步该做什么 | 中 |
| 进度感知弱 | 人生推演过程缺乏进度感 | 中 |

### 1.3 传播分享问题
| 问题 | 具体表现 | 严重程度 |
|------|---------|---------|
| 分享卡片简陋 | 结局页分享区域设计粗糙 | 高 |
| 缺乏截图生成 | 无法生成精美的分享图片 | 高 |
| 无社交元素 | 没有好友对比、排行榜展示 | 中 |

---

## 二、优化设计目标

1. **打造沉浸式体验** - 让玩家真正感受到"重开人生"的仪式感
2. **强化传播属性** - 结局截图必须让人有分享欲望
3. **提升复玩动力** - 通过视觉反馈和成就展示驱动再次游玩
4. **适配多端场景** - 微信内、浏览器、手机端都有良好体验

---

## 三、全新视觉风格方案

### 3.1 设计方向："命运卡牌"主题

将人生模拟包装为"抽取命运卡牌"的概念，整体视觉围绕**卡牌、星盘、命运**三个关键词展开。

### 3.2 色彩系统重构

```css
:root {
  /* 主色调 - 命运紫 */
  --primary: #6C5CE7;
  --primary-light: #A29BFE;
  --primary-dark: #5B4BC4;
  
  /* 辅助色 - 生命绿/警示红 */
  --success: #00B894;
  --danger: #FF7675;
  --warning: #FDCB6E;
  
  /* 背景层次 */
  --bg-deep: #0F0F1A;      /* 最深背景 */
  --bg-base: #1A1A2E;      /* 主背景 */
  --bg-surface: #16213E;   /* 卡片背景 */
  --bg-elevated: #1E2A4A;  /* 悬浮层 */
  
  /* 文字层次 */
  --text-primary: #FFFFFF;
  --text-secondary: #B8B8D1;
  --text-muted: #6C6C8C;
  
  /* 特效色 */
  --glow-primary: rgba(108, 92, 231, 0.4);
  --glow-gold: rgba(253, 203, 110, 0.3);
}
```

### 3.3 字体方案

| 用途 | 字体 | 说明 |
|------|------|------|
| 标题/数字 | "DIN Alternate", "Arial Black", sans-serif | 粗壮有力，适合大数字显示 |
| 正文 | "PingFang SC", "Microsoft YaHei", sans-serif | 清晰易读 |
| 装饰文字 | 手写体/书法字体 | 用于特殊标题，增加人文感 |

### 3.4 背景效果

**星空粒子背景**（首页/结局页）
- 使用Canvas绘制缓慢移动的星空粒子
- 鼠标移动时粒子有轻微跟随效果
- 营造"命运宇宙"的氛围感

**渐变流光边框**（天赋卡牌）
- 传说品质卡牌带有彩虹流光边框动画
- 史诗品质带有紫色脉冲光效
- 普通品质为静态边框

## 五、核心交互动效规范

### 5.1 页面转场
```css
/* 页面切换：淡入 + 上滑 */
.page-enter {
  animation: pageEnter 0.4s cubic-bezier(0.16, 1, 0.3, 1);
}
@keyframes pageEnter {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}

/* 页面退出：淡出 */
.page-exit {
  animation: pageExit 0.2s ease;
}
@keyframes pageExit {
  from { opacity: 1; }
  to { opacity: 0; }
}
```

### 5.2 按钮交互
```css
/* 默认状态 */
.btn {
  transition: all 0.2s ease;
}

/* 悬停 */
.btn:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 25px rgba(108, 92, 231, 0.3);
}

/* 按下 */
.btn:active {
  transform: translateY(0) scale(0.98);
}

/* 禁用 */
.btn:disabled {
  opacity: 0.5;
  cursor: not-allowed;
  transform: none;
}
```

### 5.3 数值变化反馈
```css
/* 属性增加 - 绿色上浮 */
.value-up {
  animation: valueUp 0.8s ease forwards;
  color: var(--success);
}
@keyframes valueUp {
  0% { transform: translateY(0); opacity: 1; }
  100% { transform: translateY(-20px); opacity: 0; }
}

/* 属性减少 - 红色下沉 */
.value-down {
  animation: valueDown 0.8s ease forwards;
  color: var(--danger);
}
@keyframes valueDown {
  0% { transform: translateY(0); opacity: 1; }
  100% { transform: translateY(20px); opacity: 0; }
}
```

## 七、响应式适配方案

### 7.1 断点设计
| 设备 | 宽度 | 适配策略 |
|------|------|---------|
| 手机竖屏 | < 480px | 单列布局，全宽卡片 |
| 手机横屏/平板 | 480-768px | 双列布局，缩小间距 |
| 平板/小屏PC | 768-1024px | 三列布局，固定侧边栏 |
| PC大屏 | > 1024px | 居中容器，最大宽度800px |

### 7.2 触摸优化
- 按钮最小点击区域：44x44px
- 滑动切换页面支持
- 长按显示详情提示

---

## 八、性能优化建议

1. **动画性能**
   - 使用transform和opacity进行动画（GPU加速）
   - 避免触发布局重排（layout thrashing）
   - 使用will-change属性提示浏览器优化

2. **资源加载**
   - 字体使用font-display: swap
   - 图片懒加载
   - CSS/JS文件压缩合并

3. **Canvas优化**
   - 星空背景使用requestAnimationFrame
   - 粒子数量控制在200以内
   - 离屏Canvas预渲染静态元素
